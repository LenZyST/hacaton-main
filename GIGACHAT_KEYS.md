# Как получить ключ GigaChat и куда его положить

## Шаг 1. Регистрация и доступ к GigaChat

1. Откройте в браузере: **https://developers.sber.ru/**
2. Нажмите **«Войти»** и авторизуйтесь через **Сбер ID** (или зарегистрируйтесь).
3. Перейдите в раздел **GigaChat API** (или по прямой ссылке: https://developers.sber.ru/studio/workspaces/my-space/get/gigachat-api ).

## Шаг 2. Создание проекта и получение ключа

1. В личном кабинете нажмите **«Создать проект»** (или откройте существующий).
2. Укажите название проекта (например, «Городской портал») и примите условия.
3. В разделе **AI-модели** выберите **GigaChat API**.
4. В настройках проекта найдите блок **«Авторизация»** / **«Ключ авторизации»**.
5. Нажмите **«Сгенерировать ключ»** (или «Скопировать ключ»).  
   Вам покажут:
   - **Client ID** (идентификатор)
   - **Client Secret** (секретный ключ)  
   Либо сразу готовую строку **Authorization** (уже в Base64) — её можно использовать как есть.

Если даны только **Client ID** и **Client Secret**, ключ в Base64 нужно собрать самим (см. шаг 3).

## Шаг 3. Формирование значения GIGACHAT_CREDENTIALS

Ключ для приложения — это строка **Base64** от `Client_ID:Client_Secret` (без пробелов).

**Вариант А.** В личном кабинете скопировали уже готовый ключ (одну строку) → это и есть значение для `GIGACHAT_CREDENTIALS`.

**Вариант Б.** Есть только Client ID и Client Secret:

В терминале (Linux/macOS):

```bash
echo -n "ВАШ_CLIENT_ID:ВАШ_CLIENT_SECRET" | base64
```

Подставьте вместо `ВАШ_CLIENT_ID` и `ВАШ_CLIENT_SECRET` реальные значения (без кавычек, без пробелов). Вывод команды — это и есть значение `GIGACHAT_CREDENTIALS`.

В Python (одноразово, чтобы получить строку):

```python
import base64
s = base64.b64encode(b"ВАШ_CLIENT_ID:ВАШ_CLIENT_SECRET").decode()
print(s)
```

## Шаг 4. Куда положить ключ

### Способ 1. Файл `.env` (удобно для разработки)

1. В корне проекта **Backend** (рядом с `main.py`) создайте файл **`.env`**.
2. Откройте его и добавьте одну строку (подставьте свой ключ):

```env
GIGACHAT_CREDENTIALS=полученная_строка_base64
```

3. Сохраните файл.  
4. Установите загрузчик переменных окружения (если ещё не установлен):

```bash
pip install python-dotenv
```

5. При запуске приложения (`python main.py` или `uvicorn main:app ...`) переменная `GIGACHAT_CREDENTIALS` подхватится из `.env` автоматически (в `main.py` уже добавлена загрузка `dotenv`).

**Важно:** файл `.env` не коммитьте в git. Он уже должен быть в `.gitignore`. Пример настроек есть в `.env.example`.

### Способ 2. Переменная окружения в терминале

Перед запуском приложения выполните (подставьте свой ключ):

**Linux/macOS:**

```bash
export GIGACHAT_CREDENTIALS="полученная_строка_base64"
python main.py
```

**Windows (cmd):**

```cmd
set GIGACHAT_CREDENTIALS=полученная_строка_base64
python main.py
```

**Windows (PowerShell):**

```powershell
$env:GIGACHAT_CREDENTIALS="полученная_строка_base64"
python main.py
```

## Шаг 5. Проверка

1. Установите SDK: `pip install gigachat`
2. Запустите бэкенд (с настроенным `GIGACHAT_CREDENTIALS`).
3. Отправьте новое обращение через форму на фронтенде.
4. В ответе/в карточке заявки должна подставиться категория, выбранная GigaChat. В БД в поле `routing_debug` при успешном ответе ИИ будет префикс `ai:`.

Если ключ не задан или неверный, распределение по категориям будет идти по старым правилам (regex), без ошибки.

## Кратко

| Что нужно | Где взять |
|-----------|-----------|
| Ключ (Base64) | developers.sber.ru → GigaChat API → проект → «Ключ авторизации» / Client ID + Client Secret → Base64 |
| Куда положить | В `.env`: `GIGACHAT_CREDENTIALS=...` или в переменную окружения перед запуском |
