# Тепловая карта и единые адреса

Чтобы на карте отображались обращения по адресам и одинаковые места не дублировались, адреса нужно **нормализовать** и получить **координаты**.

## Два подхода

### 1. API Яндекс Карт (рекомендуется)

**Плюсы:** один и тот же адрес всегда даёт один и тот же результат; подсказки и геокодирование по России работают хорошо; на карте можно сразу показывать точки.

**Как:** при сохранении обращения (или при построении карты) отправлять текст адреса в [Яндекс Геокодер](https://yandex.ru/dev/maps/geocoder/). В ответ приходят:
- нормализованный адрес (единый вид для «ул. Пушкина, 3» и «Пушкина д.3»);
- широта и долгота.

Дальше в БД хранить нормализованный адрес и координаты (или только координаты и подпись). Тепловая карта строится по точкам с координатами; «одинаковые» адреса автоматически сливаются в одну точку (или близкие точки можно схлопывать).

**Что нужно:** API-ключ в [кабинете разработчика Яндекса](https://developer.tech.yandex.ru/). Добавьте в `.env` переменную `YANDEX_GEOCODER_API_KEY=ваш_ключ`. При сохранении обращения с заполненным полем «Адрес» бэкенд вызывает Геокодер и сохраняет нормализованный адрес и координаты.

### Если приходит 403 Invalid api key

Ключ должен быть **именно для HTTP Геокодера**. Ключи от других сервисов (только JavaScript API, Статический API и т.п.) не подходят.

1. Зайдите в [developer.tech.yandex.ru](https://developer.tech.yandex.ru/) → «Подключить API» / «Сервисы».
2. Выберите сервис **«JavaScript API и HTTP Геокодер»** (в названии обязательно должно быть «HTTP Геокодер»).
3. Создайте новый ключ для этого сервиса и скопируйте его в `.env`:  
   `YANDEX_GEOCODER_API_KEY=скопированный_ключ`
4. Без кавычек, без пробелов в начале/конце. После создания ключ может активироваться до 15 минут.

### Если «Network is unreachable» или сеть недоступна

Ошибка значит, что с машины, где запущен бэкенд, нет выхода в интернет до `geocode-maps.yandex.ru` (нет сети, файрвол блокирует, или хост в изолированной среде). Обращение при этом **сохраняется**, просто без координат. Что проверить: доступ в интернет с сервера (`ping geocode-maps.yandex.ru` или `curl -I https://geocode-maps.yandex.ru/`), настройки файрвола и прокси. На локальной разработке убедитесь, что компьютер в сети и нет блокировки исходящих HTTPS-запросов.

### 2. ИИ для «похожего» адреса

**Плюсы:** можно пытаться определять похожие формулировки без геокодирования.

**Минусы:** для карты координаты всё равно нужны — без геокодера или другой привязки к месту тепловую карту не построить. ИИ может помочь сгруппировать тексты адресов, но финальный шаг — «эта группа = одна точка на карте» — всё равно требует геокодирования (хотя бы одного адреса из группы).

**Вывод:** ИИ имеет смысл как дополнение (например, подсказка «похоже на уже существующий адрес»), а за единый вид адреса и координаты лучше отвечает API Яндекс Карт.

---

## Рекомендация

Использовать **Яндекс Геокодер**:

1. В форме обращения уже есть необязательное поле «Адрес».
2. При сохранении обращения (или по кнопке «Обновить координаты» в админке) вызывать Геокодер с этим адресом.
3. В БД хранить: исходный адрес, нормализованный адрес (из ответа API), широту, долготу.
4. Тепловая карта в админке: брать обращения с заполненными координатами и рисовать **тепловой слой** (Яндекс.Карты JavaScript API 2.1 + плагин Heatmap), в стиле тепловизора — градиент от «холодных» к «горячим» зонам.

Так адреса будут приводиться к одному виду, а карта — строиться по реальным координатам.

## Миграция БД для координат (когда будете подключать Геокодер)

```sql
ALTER TABLE appeals ADD COLUMN IF NOT EXISTS address TEXT;
ALTER TABLE appeals ADD COLUMN IF NOT EXISTS address_normalized TEXT;
ALTER TABLE appeals ADD COLUMN IF NOT EXISTS lat DOUBLE PRECISION;
ALTER TABLE appeals ADD COLUMN IF NOT EXISTS lon DOUBLE PRECISION;
```

После этого бэкенд может при сохранении обращения вызывать Геокодер и записывать `address_normalized`, `lat`, `lon`.

## Карта в админке (Яндекс.Карты вместо Leaflet)

В разделе «Тепловая карта» используется **Яндекс.Карты JavaScript API 2.1** и плагин [mapsapi-heatmap](https://github.com/yandex/mapsapi-heatmap): отображается тепловой слой в стиле тепловизора (синий → голубой → зелёный → жёлтый → красный).

**Ключ для карты (фронтенд):** в корне фронтенда (или в корне проекта, если Vite читает оттуда) создайте или отредактируйте `.env` и добавьте переменную с ключом **JavaScript API** (тот же ключ, что и для Геокодера, подойдёт, если при создании ключа выбран сервис «JavaScript API и HTTP Геокодер»):

```env
VITE_YANDEX_MAPS_API_KEY=ваш_ключ
```

После изменения `.env` перезапустите фронтенд (`npm run dev`). Если ключ не задан, в блоке тепловой карты будет показана подсказка с этой инструкцией.
